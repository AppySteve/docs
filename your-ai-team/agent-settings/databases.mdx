Databases give your agents structured data to work with — tables, records, and fields they can query, update, and reference.

## How Databases Work

Unlike knowledge files (which are reference documents), databases provide structured data agents can actively query and update. Think spreadsheets your agents can read and write to.

When an agent needs to look up a customer, check inventory, or update a project status, it queries the database just like you would search a spreadsheet. The difference is your agents can do this instantly, in the middle of a conversation.

## What You Can Do With Databases

Store customer lists, track projects, maintain inventories, or any structured data your agents need. Agents can look up records, add new entries, and update existing ones.

Databases shine when you need your agents to work with information that changes regularly. Customer contact details, order statuses, appointment schedules — anything that needs to stay current and accessible.

## Setting Up a Database

Configure databases through agent settings on the web platform. Create tables to organize your data, define fields for each type of information, and import existing data if you have it.

You can start with a simple structure and add complexity as needed. Begin with basic fields like name, email, and status, then add custom fields specific to your business.

## Real-World Examples

Here's how different businesses use databases with their agents:

**A sales team** might track leads with fields for contact info, deal stage, last contact date, and notes. The agent can look up any lead instantly and update the status after each interaction.

**A service business** might maintain a client database with service history, preferences, and upcoming appointments. When a client reaches out, the agent has full context immediately.

**An e-commerce business** might store product inventory with SKUs, quantities, prices, and supplier information. The agent can check stock levels and update quantities as orders come in.

<Info>
Databases work best for information that changes frequently or needs to be queried. For static reference material, use Knowledge files instead.
</Info>
