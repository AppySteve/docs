# Agent Settings

Agent settings control the core properties of an individual agent — its name, model, and operational parameters.

## What Agent Settings Cover

Agent settings define the identity and capabilities of a single agent. This includes the agent's name, the AI model powering it, and operational parameters that affect how it performs.

These settings are specific to one agent in your roster. They don't affect other agents or workspace-wide behavior.

## Configuring Agent Settings

You can access agent settings through the web platform. Each agent has its own settings page where you can adjust these properties.

Navigate to your agent roster, select the agent you want to configure, and open its settings panel.

## Key Settings

Here are the main settings you can configure for each agent:

**Agent name** — What the agent is called in your roster. This helps you identify and organize your agents.

**AI model** — Which model powers the agent. This affects the agent's capability and speed. Different models have different strengths.

**Operational parameters** — Additional settings that control how the agent behaves and performs tasks.

<Info>
Most users don't need to change agent settings. The defaults work well for typical use. These settings are here for power users who want fine-grained control.
</Info>
