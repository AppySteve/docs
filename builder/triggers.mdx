Triggers control how and when your agents start working. Think of them as different entry points — some let users start conversations on demand, others run automatically on a schedule, and some connect to external systems that kick off work when something happens.

## How triggers work

Each trigger type is a different way to activate your agent. You can enable the ones you need and ignore the rest. Most agents use multiple triggers — for example, an agent might respond to messages on demand but also run a weekly summary automatically.

The key is choosing triggers that match how you want your agent to work. Enable what you need, mix and match, and your agent will be ready when the work comes in.

## Types of triggers

### On demand

This is the default trigger that lets users message your agent anytime through Slack or the web interface. It's perfect for conversational agents where users come with questions or need help with something specific.

Most agents use this trigger since it allows for natural back-and-forth conversations whenever users need assistance.

### Recurring schedule

Set your agent to run automatically at regular intervals. This is perfect for routine tasks like monitoring, reporting, or maintenance work that needs to happen on a predictable schedule.

You can configure:
- **Date & time** — When to start the recurring schedule
- **Frequency** — Daily, weekly, or monthly execution
- **Timezone** — Automatically uses your account timezone

**Example:** A business coach might set up a weekly check-in agent that runs every Monday morning to send personalized progress updates to clients.

### HTTP / Webhook

Let external systems trigger your agent by sending an HTTP POST request. This is perfect for connecting your agent to other tools like Zapier, Make, CRMs, or custom scripts.

When you enable this trigger, you get a unique webhook URL. Any system that can send a POST request can trigger your agent by hitting that URL with a task in the request body.

**Example:** A marketing agency might connect their CRM to trigger a lead qualification agent automatically whenever a new contact is added.

**Webhook format:**
```
POST https://builder.appy.ai/api/trigger/http/{unique-id}
Content-Type: application/json

{
  "task": "Your task here"
}
```

You can set a **Default Task** that runs when no task is included in the request body. This gives you a fallback so your agent always knows what to do.

## Real-world examples

Here's how different businesses combine triggers:

**Marketing agency** — On Demand (for client questions) + Webhook (triggered from CRM when new leads come in)

**Business coach** — On Demand (for client conversations) + Recurring Schedule (weekly check-ins every Monday)

**SaaS company** — Webhook (triggered from support system when tickets are created) + On Demand (for manual follow-ups)

The best setup depends on how you work and when your agent needs to jump in.

## Configuring triggers

You can configure triggers through the web platform's agent settings:

1. Navigate to your agent's settings
2. Click **Triggers** in the left navigation
3. Toggle each trigger type on or off
4. For recurring schedules, set the date, time, and frequency
5. For webhooks, save to generate your unique URL
6. Click **Save** to apply your changes

<Info>
After enabling the webhook trigger and saving, use the Test Trigger button to send a test request and verify everything is working correctly.
</Info>

## Getting started

Here's how to set up triggers for your agent:

1. **Start with On Demand** — Most agents need this for basic user interaction
2. **Add Recurring Schedule if needed** — Enable this for routine tasks that should happen automatically
3. **Enable Webhook for integrations** — Turn this on if you want external systems to trigger your agent
4. **Test your setup** — Use the preview or test trigger button to make sure everything works as expected

Choose the triggers that match how your users want to interact with your agent and how you want work to flow.
