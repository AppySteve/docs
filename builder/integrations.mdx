Integrations connect your agents to the tools your business already uses. Instead of just answering questions, your agents can read from and write to external services — pulling data, triggering actions, and keeping everything in sync.

## How integrations work

When you enable an integration, your agent gains the ability to interact with that external service. It can access data from connected tools, take actions in those services, and keep information synchronized across platforms.

You configure which integrations your agent can use through the agent settings on the web platform. Once configured, your agents automatically use these integrations when they need to complete tasks.

## Connecting an integration

To connect an integration to your agent:

1. Navigate to your agent's settings on the web platform
2. Click **Integrations** in the left navigation
3. Select the integration you want to add
4. Authorize the connection (you may need to log in to the external service)
5. Your agent will start using the integration automatically

Once connected, your agent knows when to use each integration based on the task at hand. You don't need to tell it when to use Gmail versus Google Calendar — it figures that out based on what the user needs.

## What agents can do

With integrations enabled, your agents can:

**Access data from connected tools** — Read emails, check calendar availability, pull records from databases, or retrieve information from any connected service.

**Take actions in external services** — Send emails, create calendar events, update database records, post messages, or perform any action the integration supports.

**Keep information in sync across platforms** — Update multiple systems at once, ensuring data stays consistent wherever it lives.

The specific capabilities depend on which integrations you enable and what actions those services support.
